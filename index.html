iOS加载bmp位图

发表于 2018-05-19 |

从网络加载bmp图片

iOS加载网络图片一般会使用SDWebImage，对于jpg、png这种常见的图片格式，SDWebImage没有什么问题，最近项目中服务端存放的是.bmp位图，SDWebImage就不好使了，报错如下

于是乎我想到了用AFNetworking下载图片

AFHTTPSessionManager *sessionManager = [AFHTTPSessionManager manager];
sessionManager.responseSerializer.acceptableContentTypes = [NSSet setWithObjects:@"image/bmp", @"image/jpg", @"image/png", @"image/jpeg", @"image/gif", nil];
sessionManager.responseSerializer = [AFImageResponseSerializer serializer];
sessionManager.requestSerializer = [AFHTTPRequestSerializer serializer];
NSURLRequest *request = [NSURLRequest requestWithURL:[NSURL URLWithString:imageUrl]];
NSURLSessionDataTask *task = [sessionManager dataTaskWithRequest:request uploadProgress:^(NSProgress * _Nonnull uploadProgress) {

} downloadProgress:^(NSProgress * _Nonnull downloadProgress) {

} completionHandler:^(NSURLResponse * _Nonnull response, id  _Nullable responseObject, NSError * _Nullable error) {
    imageView.image = responseObject;
}];
[task resume];

这里需要注意，需要设置sessionManager.responseSerializer.acceptableContentTypes，否则同样无法获取到图片数据，sessionManager.responseSerializer需要设置为AFImageResponseSerializer，这样在请求完成回调中的responseObject直接就是一个UIImage对象，可以直接给imageView.image赋值

在hexo博客中插入本地图片的方法

发表于 2018-04-13 |

使用hexo搭建博客，生成博文之后文章中的图片却只能加载网络图片？
为什么不能加载hexo网站目录内的图片呢？于是乎…某度之后，有了本文，
这位同学Nuub的博客很好的解决了我的这个问题

整体步骤总结如下：

  1. 打开博客网站根目录下的配置文件_config.yml，并修改
    post_asset-folder: true

  1. 在终端cd到博客目录下，运行npm install hexo-asset-image –save安装插件

  2. 执行hexo n xxxx之后会在source/_post目录下生成名为xxxx.md和一个同名的文件夹，之后要插入文章的图片就是要放在这个同名文件夹中

  3. 在需要引入图片的地方使用如下方法

    ![图片替换文字](文章名称/图片名称.png)

  4. 最后检查一下，hexo g生成页面后，进入public\2017\02\26\index.html文件中查看相关字段，可以发现，html标签内的语句是<img src="2017/02/26/xxxx/图片名.jpg">，而不是<img src="xxxx/图片名.jpg>。这很重要，关乎你的网页是否可以真正加载你想插入的图片。

总结结束。

ReactNative布局学习--Flex Box

发表于 2018-04-05 |

学习网址

基本概念

一、容器属性

ReactNative中flexDirection决定了主轴的方向
column为垂直方向，row为水平方向

1.1 flex-direction属性

flex-direction属性决定主轴的方向

  • row 主轴为水平方向，容器中各组件从左到右排列
  • row-reverse 主轴为水平方向，容器中各组件从右到左排列
  • column 主轴为垂直方向，各组建从上到下排列
  • column-reverse 主轴为垂直方向，个组件从下到上排列

flexDirection为row时，justifyContent指定容器内水平方向项目的对齐方式，alignItems指定容器内垂直方向上项目的对齐方式

flexDirection为时column时，justifyContent指定容器内垂直方向项目的对齐方式

1.2 flex-wrap属性

  • wrap 换行，第一行在上方
  • no-wrap 不换行
  • wrap-reverse 换行，第一行在下方（ReactNative不支持该值）

1.3 flex-flow

flex-flow是flex-direction和flex-wrap的简写形式

1.4 justify-content

justify-content 定义了项目在主轴上的对齐方式

1.5 align-items

align-items定义了项目在交叉轴上的对齐方式

1.6 align-content

属性定义了多根轴线的对齐方式。如果项目只有一根轴线，该属性不起作用。

二、项目的属性

项目的属性

2.1 order属性（ReactNative不支持）

order 定义项目的排列顺序，数值越小越排列靠前

2.2 flex-grow属性

如果所有项目的flex-grow属性都为1，则它们将等分剩余空间（如果有的话）。如果一个项目的flex-grow属性为2，其他项目都为1，则前者占据的剩余空间将比其他项多一倍。

2.3 flex-shrink属性

flex-shrink属性定义了项目的缩小比例，默认为1，即如果空间不足，该项目将缩小。

如果所有项目的flex-shrink属性都为1，当空间不足时，都将等比例缩小。如果一个项目的flex-shrink属性为0，其他项目都为1，则空间不足时，前者不缩小。
负值对该属性无效。

2.4 align-self属性

对应ReactNative中alignSelf属性，align-self属性允许单个item使用与其他项目不一样的对齐方式，可以覆盖align-items，默认值为auto，表示继承父元素的align-items属性

最近半年的工作总结

发表于 2017-10-24 |

2017年2月13号入职的这家公司，工作到现在，总共8个月的时间，从事iOS软件开发工作

工作内容

来到公司之后的主要工作是对之前项目的重构，由于之前项目各种闪退，各种bug，所以索性直接对照着功能点对该项目进行重写，复用可以复用的功能方法，完成了第一版的项目开发，之后由于BOSS对于项目的各种提要求，变需求，我们便开始了一直持续到现在的需求讨论外加demo实现验证阶段，工作的节奏基本上就是这样的：

大概两周左右向BOSS进行一次工作汇报，然后演示一下这段时间的工作成果（即做出来点什么东东，就是看看APP，然后说说之后要干嘛，吹吹牛比），我们演示一般只占用工作汇报的十分之一时间，剩下的十分之九的时间由BOSS提出新的需求和改进建议，然后我们在之后的时间里去把这些需求和建议实现到项目中去。

总结

技术方面：对蓝牙、数据库、网络状况判断、文件下载等方面的知识进行了较为深入的研究
个人认知：认识到了环境不仅可以影响一个人，环境更造就了一大批人

展望

希望现做的这个项目在明年年初可以上线运营，也算是对今年一年的工作的一个交代，另外，希望在今后的日子里自己能够抓住机会，好好努力，变得更强。
